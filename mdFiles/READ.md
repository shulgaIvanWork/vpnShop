Итоговая архитектура проекта
Проект: Автоматизированная продажа VPN-подписок через MAX бот (форк 3xui-shopbot)
Платформа: MAX (мессенджер) + 3X-UI + ЮKassa + DeepSeek AI

1. Цели (финальные)
Полностью автоматизировать продажу личных VPN-подписок:
299₽/мес, 799₽/3 мес, 1499₽/6 мес.
Внедрить реферальную систему:
Для физических лиц: за каждого приведённого пользователя → скидка 10% на следующую подписку (купон на 1 год). Суммируется до 50%.
Для компаний: за каждую приведённую компанию → скидка 10% на корпоративный тариф (на следующий платёж). Суммируется до 50%.
Организовать AI-поддержку (DeepSeek) с эскалацией владельцу (критерий неуверенности или кнопка пользователя).
Для корпоративных клиентов (9899₽/100 слотов, 19499₽/500 слотов): только личное общение; сервер создаётся и настраивается владельцем вручную, затем подключается к системе через админ-интерфейс бота.

3. Компоненты системы
3.1 MAX Bot (адаптированный 3xui-shopbot)
Основа: форк 3xui-shopbot с заменой Telegram → MAX.
Библиотека: maxapi (PyPi: pip install maxapi).
Адаптация: переписаны хендлеры, клавиатуры, сервисы под асинхронный API MAX.
Мультисерверность: поддержка нескольких панелей 3X-UI (личные + корпоративные).
3.2 Платёжный модуль (ЮKassa + YooMoney)
Интеграция из штатной 3xui-shopbot.
Библиотеки: yookassa (официальный SDK) + yoomoney (официальный SDK).
Процесс: создание платежа → получение confirmation_url → webhook → активация подписки.
Подписки (recurring) не используются – только разовые платежи.
Пользователь выбирает способ оплаты при оформлении: ЮKassa или YooMoney.
3.3 База данных и кэш
PostgreSQL – основное хранилище (пользователи, подписки, платежи, рефералы, купоны, корпоративные серверы).
Redis – кэш, состояния сессий, временные данные.
3.4 Управление клиентами 3X-UI
Библиотека: client3x (асинхронный режим).
При покупке личной подписки:
Создаётся клиент с limitIp = 1, expiryTime = now + срок.
Фоновый воркер (cron) ежедневно:
Блокирует истёкших клиентов через client.remove_client().
Отправляет уведомления пользователю за N дней до окончания (через бота).
3.5 AI поддержка (DeepSeek) с эскалацией
Пользователь пишет вопрос → бот отправляет запрос в DeepSeek API с системным промптом:
text
Ты – поддержка VPN-сервиса. Если вопрос касается оплаты, подключения, настроек – отвечай по инструкции.
Если вопрос не связан с темой, содержит угрозы, или ты не уверен – ответь только: "НЕ УВЕРЕН"
Если ответ содержит "НЕ УВЕРЕН" или оценивается низкая уверенность (по logprobs или парсингу):
Бот уведомляет владельца в личный диалог MAX с текстом вопроса и кнопкой «Ответить».
Владелец пишет ответ → бот пересылает его пользователю с пометкой «Ответ от основателя».
Пользователь может нажать кнопку «Эскалировать» при любом ответе AI, чтобы позвать владельца.
3.6 Реферальная система
Для физических лиц:
Каждый пользователь получает реферальную ссылку https://maxbot.com/ref?code=XXX.
При переходе нового пользователя связь фиксируется в таблице referrals со статусом pending.
После первой успешной оплаты приглашённым:
Пригласившему создаётся купон на скидку 10% (действителен 1 год, одноразовый).
Купон можно применить при следующей оплате подписки (вводом кода).
Скидка суммируется до 50%: при приведении 5 друзей пользователь получает 5 купонов и может применить их последовательно (или разрешён ввод нескольких купонов на один платёж до 50%).
Для компаний:
Компания (юридическое лицо) не имеет автоматической ссылки, но может рекомендовать другую компанию владельцу.
Владелец вручную (команда /grant_corp_discount <user_id>) начисляет скидку 10% на следующий корпоративный платёж компании-рефереру.
Скидка суммируется до 50% (каждая рекомендация +10%).
3.7 Корпоративные тарифы – ручное создание с подключением к системе
Механика для клиента:
Пользователь выбирает корпоративный тариф в боте → бот показывает контакт владельца (кнопка «Связаться»).
Всё дальнейшее общение – личное (владелец обсуждает требования, выбирает тариф, выставляет счёт).
Действия владельца (админ-интерфейс в MAX боте):
Вручную создаёт VPS, устанавливает 3X-UI, настраивает inbound.
Создаёт нужное количество клиентов (100 или 500) с лимитами.
Выполняет команду /add_corp_server и в интерактивном режиме указывает:
IP панели
API-ключ (из 3X-UI)
Количество слотов (100/500)
ID пользователя-клиента (из таблицы users бота)
Бот проверяет доступ к панели и сохраняет запись в таблицу corporate_servers.
Клиент может запросить информацию о своём сервере командой /my_corp_server (IP, срок действия, статус).
Воркер раз в сутки проверяет истечение корпоративной подписки (если оплачена на срок) и уведомляет владельца и клиента.

4. Модель данных (PostgreSQL)
sql
-- Пользователи
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    max_user_id BIGINT UNIQUE,
    username VARCHAR,
    subscription_end TIMESTAMP,          -- для личных подписок
    device_limit INT DEFAULT 1,
    uuid VARCHAR,
    assigned_server VARCHAR,             -- IP панели 3X-UI
    created_at TIMESTAMP DEFAULT NOW()
    -- поля для скидок не хранятся (используются купоны)
);


-- Платежи
CREATE TABLE payments (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    amount DECIMAL,
    currency VARCHAR DEFAULT 'RUB',
    status VARCHAR(20) CHECK (status IN ('pending','succeeded','cancelled')),
    yookassa_payment_id VARCHAR,
    plan_duration INT,                   -- дней
    discount_applied INT DEFAULT 0,      -- процент скидки, если использован купон
    coupon_code VARCHAR(50) NULL,
    created_at TIMESTAMP DEFAULT NOW()
);


-- Реферальные связи
CREATE TABLE referrals (
    id SERIAL PRIMARY KEY,
    referrer_id INT REFERENCES users(id),
    referred_id INT REFERENCES users(id),
    referred_type VARCHAR(10) DEFAULT 'individual', -- 'individual' or 'company'
    reward_issued BOOLEAN DEFAULT FALSE,
    coupon_code VARCHAR(50),              -- для физ. лиц
    created_at TIMESTAMP DEFAULT NOW()
);


-- Купоны (для физ. лиц)
CREATE TABLE coupons (
    code VARCHAR(50) PRIMARY KEY,
    user_id INT REFERENCES users(id),
    discount_percent INT CHECK (discount_percent <= 50),
    valid_until TIMESTAMP,
    used BOOLEAN DEFAULT FALSE,
    used_at TIMESTAMP
);


-- Корпоративные серверы (ручное добавление)
CREATE TABLE corporate_servers (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),     -- клиент-владелец сервера
    server_ip INET,
    api_key_encrypted TEXT,               -- зашифрованный API-ключ 3X-UI
    slots_total INT,                      -- 100 или 500
    slots_used INT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'active',  -- active, expired, disabled
    notes TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

5. Основные сценарии работы
5.1 Покупка личной подписки (автоматически)
Пользователь выбирает тариф → бот спрашивает о наличии купона.
При вводе кода проверяется валидность (принадлежит пользователю, не истёк, не использован).
Бот создаёт платёж в ЮKassa или YooMoney с учётом скидки.
Оплата → webhook → бот получает подтверждение.
Бот создаёт клиента в 3X-UI (limitIp=1, expiryTime), сохраняет в БД.
Пользователь получает ключ и инструкции.
5.2 Реферальная система (физ. лица)
Пользователь A получает ссылку, делится ею.
Пользователь B переходит, регистрируется, покупает подписку.
После первого успешного платежа B:
Создаётся купон на 10% для A (срок 1 год).
A получает уведомление с кодом купона.
5.3 Реферальная система (компании)
Компания X рекомендует компанию Y владельцу (личное общение).
Владелец заключает договор с Y, затем выполняет /grant_corp_discount <user_id_X>.
При следующем выставлении счёта компании X владелец видит накопленную скидку (до 50%) и выставляет счёт с её учётом.
5.4 AI поддержка с эскалацией
Пользователь пишет вопрос → бот → DeepSeek с системным промптом.
Если ответ = "НЕ УВЕРЕН" или низкая уверенность → бот пересылает вопрос владельцу с кнопкой «Ответить».
Владелец отвечает → бот отправляет ответ пользователю с пометкой.
Пользователь может в любой момент нажать кнопку «Эскалировать» для ручного вызова владельца.
5.5 Корпоративная подписка (ручное создание + подключение)
Клиент выбирает тариф → бот показывает контакт владельца.
Владелец обсуждает детали, выставляет счёт (вне бота), получает оплату.
Владелец вручную создаёт VPS, настраивает 3X-UI, создаёт 100 или 500 клиентов.
Владелец выполняет /add_corp_server и подключает сервер к системе (указывает клиента).
Клиент может проверить статус через /my_corp_server.

7. Безопасность и масштабирование
Все секреты только в .env, не в коде.
Webhook ЮKassa проверяется по подписи (IP-фильтрация или проверка заголовка).
Webhook YooMoney проверяется по notification_secret.
3X-UI панели защищены паролями, закрыты нестандартными портами.
API-ключи 3X-UI для корпоративных серверов хранятся в БД зашифрованными (Fernet).
Купоны генерируются как случайные строки (secrets.token_urlsafe) – защита от подбора.
При нагрузке до 1000 активных подписок – достаточно одного сервера 3X-UI. Для роста – добавление дополнительных панелей (поддерживается мультисерверность).
